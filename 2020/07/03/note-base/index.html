    • 使用token登录解决方法

    目前大多数采用的是请求头携带Token的形式

    1. 首次登录时，后端服务器判断用户账号密码正确之后，根据用户id、用户名、定义好的密钥、过期时间生成token,返回给前端
    2. 前端拿到后端返回的token，存储在localStroage和vuex里
    3. 前端每次路由跳转，判断localStroage 有无token,没有则跳转到登录页，有则请求获取用户信息，改变登录状态
    4. 每次请求接口，在axios请求头中携带token
    5. 后端接口判断请求头有无token，没有或者token 过期，返回401
    6. 前端得到401状态码，重定向到登录页面

    • token

    token的意思是”令牌”，是用户身份的验证方式，最简单的token组成：uid(用户唯一身份标识)、time（当前时间的时间戳）、sign（签名，由token的前几位+演绎哈希算法压缩成一定长的十六进制字符串，可以防止恶意第三方拼接token请求服务器），还可以把不变的参数也放进token，避免多次查库

    • Cookie

    cookie是保存在本地终端的数据。cookie由服务器生成，发送给浏览器，浏览器把cookie以kv形式保存到某个目录下的文本文件内，下一次请求同一网站时会把该cookie发送给服务器。由于cookie是存在客户端上的，所以浏览器加入了一些限制确保cookie不会被恶意使用，同时不会占据太多磁盘空间，所以每个域的cookie数量是有限的。

    cookie的组成有：名称（key）、值（value）、有效域（domain）、路径（域的路径，一般设置为全局：“\”）、失效时间、安全标志（指定后，cookie只有在使用SSL连接时才发送到服务器（https））。下面时一个简单的使用cookie的例子：

    cookie和localstorage的区别

    共同点：都是保存在浏览器端，且同源的。localStorage和cookie是在所有同源窗口中都是共享的

    不同点：

    1. cookie数据始终在同源的http请求中携带（即使不需要）,即cookie在浏览器
    2. 和服务器来回传递。而localStorage不会自动把数据发给服务器，仅在本地保存。cookie数据还有路径（path）的概念，可以限制cookie只属于某个路劲下。存储大小限制不同，cookie数据不能超过4k，同时因为每次http请求会携带cookie，所以cookie只适合保存很小的数据，如会话标识。
    3. localStorage 虽然也有存储大小的限制，但比cookie大得多，可以达到5M或更大。
    4. 数据有效期不同，localStorage:始终有效，窗口或浏览器关闭也一直保存，因此用作持久数据；cookie只在设置的cookie过期时间之前一直有效，即使窗口或浏览器关闭。

    session介绍

    session的中文翻译是会话，当用户打开某个web应用时，便与web服务器产生一次session，服务器使用session把用户的信息临时保存在服务器上，用户离开网站后session会被销毁，这种用户信息存储方式相对于cookie来说更加安全， 可是session有一个缺陷；如果web服务器做了负载均衡，那么下个操作请求到了另一台服务器的时候session丢失。

    • HTTP和HTTPS区别

    http协议

    是一种使用明文数据传输的网络协议。一直以来htto协议都是最主流的网络协议，但是互联网发展到今天，http协议的明文传输会让用户存在非常大的安全隐患。试想一下，假如你在一个http协议的网站上购物，你需要在页面上输入你得银行卡号和密码，然后你把数据提交到服务器实现购买。假如这个适合，你得传输数据被第三者给截获了，由于http明文数据传输原因。你得银行卡号和密码，将会被这个截获人所得到。现在你还敢在一个http网站上面购物吗？你还会在一个http网站上面留下你得个人信息吗？

    https协议

    https协议可以理解为http协议的升级，就是在http的基础上增加了数据加密。 在数据进行传输之前，对数据进行加密，然后再发送到服务器。这样就算数据别第三者所截获，但是由于数据是加密的，所以你得个人信息是安全的。

    • HTML5 WebSocket（全双工通讯的协议）

      简介：
      websocket 是html5 开始提供的一种在单个TCP 连接上进行全双工通讯的协议。

    websocket使得客户端和服务器之间的数据交换变得更加简单，允许服务端主动像客户端推送数据。在websocket API 中，浏览器和服务器只需要完成一次握手，两者之间就直接可以创建持久性的连接，并进行双向数据绑定传输。

    在websocket api中，浏览器和服务器只需要做一个握手的动作，然后，浏览器和服务器之间就形成了一条快速通道。两者之间就直接可以数据互相传送。

    现在，很多网站为了实现托送技术，所用的技术都是ajax轮询。轮询是在特定的时间间隔（如1秒），由浏览器对服务器发送http请求，然后由服务器返回最新的数据给客户端的浏览器。这种传统的模式带来很明显的缺点。即浏览器需要不断向服务器发送请求，然而http请求可能包含较长的头部，期中真正有效的数据可能只是很小的一部分，显然这样会浪费很多的带宽资源

    html5 定义的websocket 协议，能更好的节省服务器资源和带宽。并且能够实时进行通讯。
